# Backlog task rules

- Work on one numbered task at a time unless the user explicitly approves parallel tasks.
- Do not mark a task complete when an acceptance criterion is deferred or unverified.
- Keep task scope stable. Add follow-up files for discovered work.
- Preserve deterministic simulation behavior and the WebGPU-only product decision.
- Human review scripts must describe observable behavior and diagnostics, not implementation details.
- Use zero-padded IDs and imperative filenames: `NNN-short-outcome.md`.
- Move completed files into `completed`; do not duplicate them or maintain a separate status database.
